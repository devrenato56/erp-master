Diagrama Entidad-Relación — Chatbot de capacitación ERP

Modelo de datos propuesto para Supabase (PostgreSQL + pgvector)

Notas de implementación:
· La tabla CHUNK usa pgvector (columna tipo vector) nativo en Supabase/PostgreSQL para almacenar los embeddings del RAG, sin necesidad de un vector store externo.
· DOCUMENTO separa visibilidad (privado/compartido, elegido por el usuario) de estado_moderacion (resultado del filtro automático vía LLM).
· EVALUACION es la plantilla; INTENTO_EVALUACION registra cada vez que un usuario la responde, permitiendo múltiples intentos por usuario.
